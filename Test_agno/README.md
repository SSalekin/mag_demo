# MAG Demo (Multi-Agent System)

This repository contains the source code for the architecture of the multi-agent system (MAG) - **Software Factory**.

## Architecture & Components (Phase 5)
The software factory is now fully modular and functional. It is driven by **Ollama** via the **Agno** framework and consists of the following components:

### The Multi-Agent Team
- **Manager**: Supervisor of the overall process. Coordinates the team and divides the work (via tool calling).
- **DevWeb**: Specialist in front-end and web development.
- **DevSoft**: Specialist in software and algorithmic development.
- **DevOps**: In charge of infrastructure and deployment.
- **Test**: Responsible for verification and execution in a Docker Sandbox.
- **Evaluation**: Final quality controller before validation.

### Custom Tools and Modules
- **Titan Memory (`memory_tool`)**: A local neural memory model (based on PyTorch) used by the Devs and the Evaluator to dynamically and scientifically memorize the context (with synaptic activation logs and surprise measurement).
- **Docker Sandbox (`sandbox_tool`)**: The Test agent has a secure execution module that completely isolates the generated code via the Docker API.
- **Workspace (`workspace_tool`)**: Agents physically save their code in the `workspace/` directory so the user can view and use it before pushing to production.

## Installation & Prerequisites

### 1. Python Environment Preparation
```bash
python -m venv venv
# Windows:
.\venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

pip install -r requirements.txt
```

### 2. Ollama Configuration
The system uses **Qwen 2.5 (3B)**, an extremely capable model for agent orchestration and Tool Calling while remaining very lightweight (ideal for small machines).
Make sure you have installed [Ollama](https://ollama.com/), then download the model:
```bash
ollama pull qwen2.5:3b
```

### 3. Docker Configuration
The Test agent requires **Docker Desktop** (or the Docker engine) to be installed and running on your machine.

## Starting the Software Factory

To launch the application and interact with the Manager via the interactive interface:

```bash
python main.py
```

### Automatically generated folders:
- `workspace/`: Contains all the source code generated by the team.
- `logs/`: Contains the complete history (e.g., `memory_tracker.log`) with scientific metrics of the Titan memory (saturation rate, LTM activations).
- A local `neural_memory.pt` file will be created at the root during the first launch to store the dynamic weights.
